2. TensorFlow Playground: The Circle Problem Case Study

To put this architecture to the test, I utilized the TensorFlow Playground to train a model capable of separating a complex, non-linear dataset (a circle within a ring). Here is the exact configuration and the insights gained from the experiment.

Experiment Configuration

Key Observations & Results

1. The Necessity of Hidden Layers: Attempting to solve the circular dataset with 0 hidden layers immediately failed. A neural network without hidden layers operates as a simple linear classifier, and it is mathematically impossible to draw a single straight line that isolates a circle. Adding hidden layers allowed the model to construct intersecting lines to form boundaries.

2. Why "4x2" Architecture Worked: By assigning 4 neurons to the first hidden layer, the network effectively learned to draw four distinct linear boundaries (forming a rough square). Passing those outputs to the second hidden layer (2 neurons) allowed the model to smooth those edges, combining them into a tight circular classification boundary.

3. The Role of Tanh Activation: Utilizing the Tanh (Hyperbolic Tangent) activation function was critical. It introduced the non-linearity required to bend the feature space, outputting values between -1 and 1, which perfectly mapped to the coordinate space of the circular dataset.

4. Learning Rate Sensitivity: At a learning rate of 0.03, the gradient descent algorithm steadily minimized the Loss Function, converging on a solution smoothly. When artificially pushed to a high rate (e.g., 3.0), the model overshot the optimal weights constantly, causing the classification background to flicker wildly without ever learning the pattern.
